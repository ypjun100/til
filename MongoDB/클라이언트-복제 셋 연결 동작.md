# 클라이언트-복제 셋 연결 동작

* 몽고DB 클라이언트 라이브러리는 독립 인스턴스, 복제 셋 관계없이 몽고DB 서버와의 통신을 관리 가능함
* 복제 셋이면 기본적으로 클라이언트는 프라이머리에 연결되고, 모든 트래픽을 프라이머리에 라우팅함
* 클라이언트가 복제 셋에 연결하기 위해서는 시드(데이터를 읽고 쓸 복제 셋의 멤버) 목록이 필요함
* 시드 목록에 모든 멤버를 나열할 필요는 없으며, 클라이언트는 시드에 연결되면 다른 멤버들을 발견함

```
mongodb://server-1:27017,server-2:27017,server-3:27017
```

* 복제 셋의 목적은 네트워크 파티션이나 서버가 다운될 때에도 데이터의 가용성을 높이는 것임
* 클라이언트가 도달할 수 없는 프라이머리가 없는 동안에는 애플리케이션이 쓰기를 수행할 수 없음
* 기본적으로 클라이언트는 이 기간 동안 요청을 보내지 않지만, 필요하다면 세컨더리로 읽기를 요청 가능함
