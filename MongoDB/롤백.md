# 롤백

```
A : 1 -> 2 -> 3 -> 4
B : 1 -> 2 -> 3' -> 4'
```

* 복제 셋 환경에서 oplog 차이로 인해 데이터 불일치가 발생할 때, 멤버가 저장한 데이터를 되돌리는 작업
* 프라이머리가 다운돼 세컨더리가 프라이머리로 선출될 때, 동기화하지 못한 상태에서 선출이 발생할 수 있음
* 프라이머리가 된 멤버(B)에 새로운 연산이 추가된 이후, 기존 프라이머리(A)가 복구되는 경우 롤백 발생
	* A 멤버가 다시 원상 복구가 되는 경우, B의 oplog를 A에 반영하는 절차가 진행됨
	* A 멤버의 3, 4는 동기화되지 못한 연산, B 멤버의 3', 4'는 B가 프라이머리가 된 이후 발생한 연산임
* 이때 A가 다시 프라이머리가 되기 위해서 3, 4 연산을 제거한 후 3', 4' 연산을 가져온 후 반영하게 됨

### 롤백의 동작 방식

1. **공통 oplog 시점 찾기**
	* 장애 복구 시 A는 B의 oplog와 자신의 oplog를 비교하여, 두 oplog가 일치하는 공통 시점을 찾음
2. **불일치 데이터 삭제 및 복원**
	* 공통 시점 이후의 oplog에 기록된 모든 작업을 취소함 (oplog 뿐만 아니라 실제 컬렉션도 반영됨)
3. **롤백 데이터 저장**
	* 롤백에서 변경된 도큐먼트는 몽고DB 내부에 BSON 파일로 저장되어, 나중에 수동으로 복구가 가능함
4. **동기화 재개**
	* 롤백이 완료되면, A 멤버는 B 멤버의 데이터를 기준으로 동기화를 재개하게 됨
