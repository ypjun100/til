# 스웜 모드 서비스

- 도커 명령어의 기본 제어 단위는 컨테이너이지만, 스웜 모드에서 기본 제어 단위는 서비스임
- 서비스는 컨테이너의 집합이며, 서비스를 제어하면 해당 서비스 내의 컨테이너에 같은 명령어가 전달됨
- 서비스 내에 컨테이너는 1개 이상 존재할 수 있으며, 이러한 컨테이너들을 태스크라고 함

> [!Question] 서비스와 태스크
> 서비스는 무엇(어떤 이미지를?)을 어떻게(레플리카를 포함해서? 등) 실행할지를 정의한 명세이며, `docker service create`로 서비스를 생성할 수 있다. 그리고 태스크는 서비스 명세에 따라 실제로 클러스터 노드에 배치되는 개별 컨테이너의 배포 단위이다. 스웜 매니저가 서비스 명세를 파악한 후, 가용한 노드에 지정된 레플리카 수만큼 태스크를 생성해 할당한다. 즉, 스웜 모드에서는 "컨테이너" 대신 "태스크"라고 표현한다.

- 레플리카를 3으로 설정하면 스웜 스케줄러는 컨테이너를 분산하기 위한 적합한 노드를 선택 및 할당함
	- 위 그림처럼 한 노드에 여러 개의 태스크가 할당될 수도 있음
- 스웜은 서비스의 상태를 확인하다가 레플리카 수만큼 태스크가 없으면, 새로운 태스크 레플리카를 생성함
- 위 그림의 경우 태스크가 할당된 노드가 다운되면 매니저는 다른 노드에 같은 태스크를 생성하게 됨
- 서비스는 컨테이너 이미지를 업데이트해야 할 때 컨테이너에 순서대로 배포하는 롤링 업데이트 기능을 제공

> [!Question] 롤링 업데이트
> 롤링 업데이트는 여러 서버, 컨테이너 등으로 구성된 클러스의 설정이나 데이터를 변경하기 위해 하나씩 재시작하는 것을 의미한다. 롤링 업데이트를 이용하면 하나를 업데이트해도 다른 서버나 컨테이너는 작동 중이기 때문에 지속적인 서비스가 가능하다.

### global 서비스 생성하기

```shell
docker service create --name global_web --mode global nginx
```

- 서비스는 레플리카셋의 수를 정의하는 복제 모드와 모든 노드에 컨테이너를 생성하는 글로벌 모드로 나눔
- 글로벌 모드는 스웜 클러스터 내에서 사용할 수 있는 모든 노드에 컨테이너를 반드시 하나씩 생성함
- 보통 스웜 클러스터를 모니터링하기 위한 에이전트 컨테이너 등을 생성해야 할 때 사용됨
- `--mode` 옵션의 값을 별도로 설정하지 않으면 기본적으로 복제 모드를 사용하게 됨
