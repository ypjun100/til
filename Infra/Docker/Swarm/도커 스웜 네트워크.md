# 도커 스웜 네트워크

- 스웜 모드는 여러 도커 엔진에 같은 컨테이너를 분산하기에 네트워크를 하나로 묶는 네트워크 풀이 필요함
- 또한, 서비스를 외부로 노출할 때 어느 노드로 접근해도 컨테이너에 접근 가능한 라우팅 기능이 필요함
- 이러한 네트워크 기능은 스웜 모드에서 제공하는 네트워크 드라이버를 통해 사용할 수 있음

### ingress 네트워크

![](https://img1.daumcdn.net/thumb/R1280x0/?scode=mtistory2&fname=https%3A%2F%2Fblog.kakaocdn.net%2Fdna%2FMllEt%2FbtrE3MicbYL%2FAAAAAAAAAAAAAAAAAAAAAH_M856R6rdo3Xd13CnFt-qGBWtQP8HGT9ZUXoHwxFZB%2Fimg.png%3Fcredential%3DyqXZFxpELC7KVnFOS48ylbz2pIh7yKj8%26expires%3D1782831599%26allow_ip%3D%26allow_referer%3D%26signature%3D4YZkIyhKdaJ1XvgBKiDzpH8eUko%253D)

- 스웜 클러스터를 생성하면 자동으로 등록되는 네트워크로, 스웜 모드를 사용할 때만 유효함
- 어떤 스웜 노드에 접근하더라도 서비스 내의 컨테이너에 접근할 수 있는 라우팅 메시를 구성함
- 또한, 서비스 내의 컨테이너에 대한 접근을 라운드 로빈 방식으로 분산하는 로드 밸런싱을 담당함
- 스웜 모드로 생성된 컨테이너에 접근하기 위해서 ingress 뿐만 아니라 직접 포트로의 접근도 가능함

### 오버레이 네트워크

![](https://img1.daumcdn.net/thumb/R1280x0/?scode=mtistory2&fname=https%3A%2F%2Fblog.kakaocdn.net%2Fdna%2FAp4wU%2FbtrFjyiGU7Q%2FAAAAAAAAAAAAAAAAAAAAAMYxbxUttqL6ly87q1Vsx6JoX5Sr2Gii7IXlBZS6t55K%2Fimg.png%3Fcredential%3DyqXZFxpELC7KVnFOS48ylbz2pIh7yKj8%26expires%3D1782831599%26allow_ip%3D%26allow_referer%3D%26signature%3DsT6PZwoWGsmOM30w99lsyjzLN14%253D)

- 오버레이 네트워크는 여러 개의 도커 데몬을 하나의 네트워크 풀로 만드는 네트워크 가상화 기술
- 도커에 오버레이 네트워크를 적용하면 여러 도커 데몬에 존재하는 컨테이너가 서로 통신할 수 있음
	- ingress 네트워크는 오버레이 네트워크 드라이버를 쓰는 내장된 overlay 네트워크

> [!Question] 왜 "overlay"라고 부를까?
> "overlay"는 물리 네트워크 위에 또 다른 가상 네트워크를 쌓았다는 의미로 해석되며, 실제로 Docker overlay 네트워크는 소프트웨어로 만들어낸 논리적 네트워크로, 컨테이너 간 가상 링크, 가상 IP, 가상 서브넷을 가지게 된다.
