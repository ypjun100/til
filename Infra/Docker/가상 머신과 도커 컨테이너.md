# 가상 머신과 도커 컨테이너

![](https://img1.daumcdn.net/thumb/R1280x0.fwebp/?fname=http://t1.daumcdn.net/brunch/service/user/9vB/image/TSWVqfJUVCi9VQYpxm-9CAgNELE)

- 기존 가상화 기술은 하이퍼바이저를 이용한 여러 OS를 하나의 호스트에서 생성하는 가상 머신 생성 방식
	- 대표적인 가상화 툴로는 VirtualBox, VMWare 등이 있음
- 하이퍼바이저에 의해 생성되는 OS는 게스트 OS라고 하며, OS별로 완전한 독립 공간을 할당받음
- 하지만 시스템 자원을 가상화, 독립화하는 작업은 하이퍼바이저를 거치기 때문에 성능 손실이 발생함

> [!NOTE]
> 하이퍼바이저는 물리 하드웨어 위에서 여러 VM을 동시에 실행할 수 있게 해주는 소프트웨어로, 하드웨어 가상화, 자원 분배 및 관리, 격리 보장을 지원한다. VM은 게스트 OS를 포함하기에 무겁지만, 커널 수준까지 완전히 격리되는 강항 보안이 장점이다.

- 컨테이너에 필요한 커널은 호스트의 커널을 공유하고, 컨테이너 안에는 앱 구동에 필요한 실행 파일만 존재함

> [!NOTE]
> 도커 컨테이너는 가상화된 공간을 생성하기 위해 리눅스의 자체 기능인 `chroot`, `namespace`, `cgroup`을 사용하여 프로세스 단위의 격리 환경을 구성한다.
> 
> - `chroot`: 프로세스의 루트 디렉토리를 변경하는 명령으로, 해당 경로 밖의 파일은 접근 불가
> - `namespace`: 프로세스마다 격리된 pid(프로세스 목록), net(네트워크 인터페이스) 등을 제공하기위한 리눅스 커널 기능
> - `cgroup`: 프로세스가 사용할 수 있는 자원의 양을 제한
