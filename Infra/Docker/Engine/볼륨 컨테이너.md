# 볼륨 컨테이너와 도커 볼륨

### 볼륨 컨테이너

```shell
# volume_container의 볼륨을 service_a 컨테이너에도 공유
docker run -it --name service_a --volumes-from volume_container ubuntu:24.04
```

- 컨테이너 생성 시 `--volumes-from` 을 설정하면, `-v` 을 적용한 컨테이너 볼륨 디렉토리를 공유 가능함
	- 이는 대상 컨테이너의 마운트 메타데이터(어떤 경로가 어디에 마운트 됐는지)를 복사하는 방식임
	- 따라서 볼륨 컨테이너가 정지 상태여도 공유는 유지됨 (마운트 정보가 남아있으므로)
- 새로 만든 컨테이너도 `-v`로 생성된 컨테이너에서 설정된 볼륨 경로와 동일한 경로에 볼륨이 연결됨

### 도커 볼륨

```shell
docker volume create --name myvolume
```

- 컨테이너가 삭제되어도 데이터를 유지하고, 컨테이너 간 데이터를 공유하기 위한 도커의 스토리지 기능
- 볼륨은 호스트 시스템의 `/var/lib/docker/volumes/` 경로에서 도커 엔진이 직접 관리함
- 볼륨 생성 시에 플러그인 드라이버를 설정해 여러 종류의 스토리지 드라이버를 사용할 수 있음
	- 기본값은 `local`이며, NFS, AWS EBS 등의 여러 플러그인을 제공함
- 도커 볼륨으로 컨테이너를 생성할 때에는 기존 `-v` 옵션을 사용하는 방식과 동일하게 사용하면 됨
	- 컨테이너에서 볼륨 내에 파일을 쓰는 경우 호스트 볼륨 경로에도 동일하게 반영됨

### Stateless 설계

- 컨테이너 내부에 상태를 저장하지 않는 설계로, 필요한 데이터는 외부 저장소에서 가져오는 방식임
	- 반대로, Stateful은 상태를 컨테이너 내의 메모리나 디스크로 저장하고 유지하는 방식임
- 컨테이너는 언제든지 교체될 수 있는 일시적 존재이기에 컨테이너 종료 시에 데이터도 함께 삭제됨
- 따라서, Stateless 설계에서는 데이터를 외부에 위임하여 데이터를 컨테이너 밖에서 보존함
