# 컨테이너 자원 할당 제한

- `run`, `create` 명령에 옵션을 입력하지 않으면, 컨테이너는 호스트의 자원을 제한 없이 쓸 수 있음
- 운영 환경의 컨테이너에는 자원을 제한해 호스트와 다른 컨테이너를 방해하지 않게 설정하는 것이 좋음

### 컨테이너 메모리 제한

```shell
docker run -d --memory="1g" nginx
```

- `docker run` 명령에 `--memory` 옵션을 지정해 컨테이너의 메모리를 제한할 수 있음
- 입력 가능 범위는 m(megabyte), g(gigabyte)이며, 위 예시에서는 컨테이너 메모리를 1GB로 제한함
- **컨테이너 내에서 동작하는 프로세스가 할당 메모리를 초과하면 컨테이너는 자동으로 종료됨**

### 컨테이너 Swap 메모리 제한

```shell
docker run -d --memory="200m" --memory-swap="500m" nginx
```

- 기본적으로 컨테이너의 Swap 메모리는 메모리의 2배로 설정되지만, 별도 옵션으로 크기를 지정할 수 있음

### 컨테이너 CPU 제한

```shell
docker run -it --cpu-shares 1024 nginx
docker run -it --cpu-shares 512 nginx
```

- `--cpu-shares` 옵션은 컨테이너가 CPU를 상대적으로 어느 비중만큼 나눠 쓸 건지를 명시하는 옵션
- 이 옵션은 상대적인 값으로 초기에 컨테이너가 가지는 값은 `1024`이며, 이는 1의 비중을 의미함
- 위 예시대로 컨테이너를 생성하면 첫 번째 컨테이너와 두 번째 컨테이너는 2:1 비율로 CPU를 공유함

### 컨테이너 CPU 개수 제한

```shell
docker run -it --cpus=0.5 nginx
```

- `--cpus`는 컨테이너가 사용할 수 있는 CPU 코어 수의 최댓값을 직접 지정하는 옵션임
	- 정수값을 사용하면 최대 N코어 사용, 소수값은 전체 CPU의 N% 사용, 0은 제한없음을 의미함

> [!TIPS]
> CPU를 많이 소모하는 워크로드를 수행해야 한다면, `--cpuset-cpu` 옵션을 사용하는 것 이 좋다. `--cpuset-cpu` 옵션은 특정 컨테이너가 특정 CPU에서만 동작하는 ***CPU 친화성(Affinity)**을 보장할 수 있고, CPU 캐시 미스 같이 성능을 하락시키는 요인을 최소할 가능성이 높아진다.
> 
> *CPU 친화성 : L1/L2 캐시는 각 코어별로 존재하고, L3/L4 캐시는 모든 코어가 공유하는 방식이기에, 만약 컨테이너에서 컨텍스트 스위칭 시에 다른 코어로 전환되면, L1/L2 캐시가 무효화(캐시 미스)되고, 새 코어는 데이터를 다시 메모리에서 가져와야 한다. 따라서, 지정된 코어만을 사용하여 해당 코어 외에는 스케줄링이 되지 않으므로 L1/L2 캐시 재사용률을 높이는 전략이다.
