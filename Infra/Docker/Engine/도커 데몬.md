# 도커 데몬

```shell
which docker
/usr/bin/docker

ps aux | grep docker
root ... /usr/bin/dockerd -H fd://...
```

- 도커는 클라이언트 기능을 수행하는 도커 클라이언트와 실제 컨테이너를 관리하는 도커 서버로 나뉨
- 도커 클라이언트는 도커 데몬에서 제공하는 API를 쉽게 사용할 수 있도록 하는 CLI 환경을 의미함
	- 사용자가 사용하는 `docker` 명령어는 도커 클라이언트를 사용하며, 이 요청은 데몬에게 전달됨
	- `/var/run/docker.sock`에 위치한 유닉스 소켓을 통해 도커 데몬의 API를 호출하게 됨
- 도커 서버는 dockerd 프로세스로서 동작하며, API를 받아 도커 엔진의 기능을 수행하는 데몬 프로세스

- 터미널에서 `docker` 명령어를 입력하면 아래와 같은 과정으로 도커가 제어됨

1. 사용자가 `docker version` 같은 도커 명령어를 입력함
2. `/usr/bin/docker`는 `/var/run/docker.sock` 유닉스 소켓을 사용해 도커 데몬에게 명령어를 전달함
3. 도커 데몬은 이 명령어를 파싱하고 명령어에 해당하는 작업을 수행함
4. 수행 결과를 도커 클라이언트에 반환하고 사용자에게 결과를 출력함

> [!Question] 유닉스 소켓이란?
> 유닉스 소켓은 같은 호스트 내의 프로세스 간 통신(IPC)을 위한 통신 엔드포인트이다. 네트워크를 거치지 않고 파일 시스템 경로를 통해 데이터를 주고 받는다. 일반 TCP 소켓은 IP 주소를 기반으로 네트워크 스택을 통해 통신하지만, 유닉스 소켓은 파일 시스템 경로(ex. `/var/run/docker.sock`)를 식별자로 사용하기에 네트워크 스택을 완전히 건너뛰기 때문에 속도가 더 빠르고 오버헤드가 낮다.
> 
> *도커 데몬의 방식 : 도커 데몬이 유닉스 소켓을 열어두고 대기하면, 도커 클라이언트가 이 소켓 파일에 HTTP 요청을 써넣고, 데몬이 읽어서 처리한 뒤 응답을 돌려준다.
