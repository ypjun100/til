# run 명령어의 옵션
### `-d` 옵션

- `run` 명령어를 사용할 때 `-it` 옵션을 사용한다면, 컨테이너를 생성하고 곧바로 Attach 모드로 접근함
	- Attach 모드는 컨테이너 내부로 쉘을 연결하는 것으로, 사용자가 컨테이너로 접근할 수 있도록 함
- 반면에 `-d` 옵션은 Detached 모드로 바로 쉘을 연결하는 것이 아닌, 백그라운드에서 컨테이너를 실행함
	- Detached 모드로 실행된 컨테이너도 이후에 `docker exec` 명령어로 컨테이너로 접속 가능함

> [!IMPORTANT]
> Detached 모드인 컨테이너는 반드시 컨테이너에서 프로그램이 실행되어야 하며, foreground 프로그램이 실행되지 않으면 컨테이너는 종료된다. foreground 프로그램은 Dockerfile의 `CMD`로 명시하거나 `docker run` 명령어 실행 시에 붙이는 프로세스가 foreground가 된다.

### `-e` 옵션

- `-e` 옵션은 컨테이너 내부의 환경변수를 설정하며, 컨테이너 내부에서 값을 가져와야 하는 경우에 사용됨

### `--link` 옵션

```
docker run --link mysql:db
```

- `--link` 옵션은 내부 IP를 알 필요없이 항상 컨테이너에 별명(alias)으로 접근하도록 설정함
- 이 옵션은 컨테이너 내부에서 호스트의 다른 컨테이너로 붙길 원할 때 사용 가능하면, 다른 컨테이너의 IP로 접속하는 것이 아닌, 호스트 명(alias로 설정한)으로 접속이 가능함
	- 위 예시는 컨테이너 내부에서 'db'라는 별명으로 mysql 컨테이너를 접속할 수 있게함
- 옵션에 명시한 다른 컨테이너가 현재 실행 중이지 않은 상태라면, 생성하는 컨테이너 또한 실행할 수 없음
