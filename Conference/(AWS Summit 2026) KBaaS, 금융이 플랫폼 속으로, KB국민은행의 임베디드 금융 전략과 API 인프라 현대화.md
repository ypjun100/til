# KBaaS, 금융이 플랫폼 속으로, KB국민은행의 임베디드 금융 전략과 API 인프라 현대화
Presenter. 김우진 매니저, KB국민은행

- 은행 인프라 내에서 보유하고 있던 데이터 & API는 이제 내부에서 외부까지 공유가 이뤄지고 있음
- **임베디드 금융** : 비금융 서비스 내에 금융 서비스를 자연스럽게 통합하는 과정으로, 별도 금융 앱이나 기관을 거치지 않고, 자연스럽게 해당 외부 앱 내에서 금융 기능을 사용할 수 있는 서비스 모델
- **BaaS** : 금융기관의 핵심 금융 기능을 API 기반으로 외부 플랫폼에 제공하여 비금융 기업이 자사 서비스 안에서 금융 기능을 구현할 수 있도록 지원하는 모델

### KB국민은행의 임베디드 금융

- 스타벅스, SSG 닷컴 등의 외부 시스템에서 국민은행의 기능을 사용할 수 있도록 임베디드 금융 기능 제공
- 임베디드 금융을 통한 파트너사 및 고객 맞춤형 상품 제공이 가능, 고객의 일상으로 파고드는 아웃바운드 제공
- 1,800여개의 API가 일별 2억개의 호출량을 가지며, 늘어나는 환경을 위한 API Gateway 인프라 구축
- 인프라를 구축하면서 동시에 안정적인 서비스 제공을 위해 아래의 최적화를 통한 혁신 진행

### 서비스 안정성

- **Graceful Shutdown**
	- 배포/업데이트, 스케일인, 장애 감지 시 파드를 Gracefully하게 종료하도록 구현
	1. 트래픽 유입 차단
	2. 종료 신호 전달
	3. Connection Draining
	4. 진행 중인 요청 완료
	5. 요청 완료 후 Pod 종료
- **배포 프로세스 최적화**
	- 블루-그린, 카나리, 롤릴 배포 방식 모두 적용하여 테스트 진행
	- 3가지 방법 중에서 최종적으로 롤링 방식을 적용 및 Graceful Shutdown으로 기존 거래 영향 없음

### 애플리케이션 성능

- **Non-Blocking Gateway(WebFlux + Netty)** : 응답을 기자리는 동안에도 다른 요청을 처리하는 논블로킹 방식으로 빠른 처리 시간 보장
- **Asynchronus Logging** : 거래 성능에 영향을 주지 않으면서 로그 유실 없이 안정한 로깅 구현
- **DB 토큰 검증 거래 성능 테스트(분산 캐시 Vs. Aurora DB + 내부 캐시(Caffeine))** : 분산 캐시 기반 구조가 가장 높은 성능을 보였지만, 비용 대비 효과와 운영 효율성을 함께 고려하여 현재는 Aurora DB와 내부 캐시 기반으로 서비스 운영중
